# PPT 单页抛光

`ppt-review` 发现某页视觉质量不达标但叙事结构无需改动时，进入本 skill 做 HTML 级视觉修复。

## 目标

在不改变页面核心叙事的前提下，修复单页视觉质量，直接修改对应的 `pages/page_XX.html`。

## 触发条件

- `ppt-review` 判定某页信息过空或信息过挤
- `ppt-review` 判定某页视觉层级不清、元素碰撞或溢出安全区
- `ppt-review` 判定某页与 `style-spec.json` 的执行偏差明显
- `ppt-review` 判定某页缺少背景装饰层或前景装饰层
- `ppt-review` 判定某页组件皮肤退化为默认白卡片
- 用户主动要求对某页做视觉调整

## 输入

- 目标页面的 `pages/page_XX.html`（必须已存在）
- `style-spec.json`（必须已存在，用于核对风格执行）
- `storyboard.json`（必须已存在，用于确认页面叙事不变）
- 可选的 `asset-plan.json`（用于确认图片槽位状态）

## 输出

- 更新后的 `pages/page_XX.html`
- 如果抛光中发现超出视觉范围的问题（如叙事缺失、资产缺失），在完成反馈中标注残留问题并建议转到对应 skill

## 可处理问题

- 信息过空：内容区域留白过多，承载不足
- 信息过挤：文字或组件堆叠，缺少呼吸空间
- 视觉层级不清：标题、正文、辅助信息之间缺少对比
- 与 `style-spec.json` 的执行偏差：配色、字体、组件皮肤未忠实消费
- 元素碰撞或安全区问题：内容侵入页脚安全区或边缘溢出
- 缺少装饰层：背景或前景 motif 缺失或退化为不可感知
- 组件皮肤退化：卡片、面板、表格等退回默认样式

## 执行规则

### 1. 依赖检查

- 确认目标页面 HTML 文件存在且可读。
- 确认 `style-spec.json` 存在且可读。
- 读取 `storyboard.json` 对应 `page_id`，确认页面叙事角色和 `payload_budget`。

### 2. 问题诊断

- 逐项检查以下清单，标记需要修复的项：
  - [ ] 页面结构层级完整：`.wrapper` > `#bg` > `#ct` > `#footer`
  - [ ] 画布尺寸严格 1280x720，无滚动条
  - [ ] 页脚安全区（右下 160x60）未被内容覆盖
  - [ ] 标题在 `#ct` 或 `#header` 内，未被其他层遮挡
  - [ ] 背景装饰层存在且带 `data-layer="bg-motif"`
  - [ ] 前景装饰层存在且带 `data-layer="fg-motif"`
  - [ ] motif 元素带有 `data-motif-key`
  - [ ] 组件皮肤与 `style-spec.json.component_skins` 一致
  - [ ] 配色与 `style-spec.json.color_roles` 一致
  - [ ] 字体策略与 `style-spec.json.typography` 一致
  - [ ] 信息密度符合 `payload_budget` 要求
  - [ ] 视觉层级清晰：标题 > 小标题 > 正文 > 辅助

### 3. 修复执行

- 按以下优先级逐项修复：
  1. 结构性问题：安全区侵犯、画布溢出、层级结构缺失
  2. 风格执行偏差：配色、字体、组件皮肤
  3. 装饰层缺失：补齐 `bg-motif` 和 `fg-motif`
  4. 密度与层级：调整留白、间距、字号对比
  5. 细节打磨：对齐、圆角、阴影、过渡一致性

### 4. 修复验证

- 修复后重新过一遍诊断清单，确认所有项通过。
- 确认修复未改变页面承载的核心内容和叙事角色。
- 确认修复后的页面与同 deck 其他页面保持同一视觉系统。

## 用户回显

- **开始**：说明当前锁定的页面和主要抛光问题。
- **完成**：概括本页做了哪些视觉调整、还有没有残留问题，以及 `下一步`。

## 关键原则

- 不要趁机改写整套叙事。
- 优先通过层级、密度、留白、视觉对比解决问题。
- 保持与同 deck 其他页面的设计语言一致。
- 必须忠实消费 `style-spec.json` 的字段，不要退回通用默认样式。
- 背景装饰层必须是用户可感知的视觉层，不能退化成微小角标或极淡纹理。
- 修复范围仅限视觉层面；如果发现叙事、资产或规划层面的问题，标注残留并建议转到对应 skill。

## 禁止事项

- 不要修改页面的核心叙事内容或 `content_blocks` 语义。
- 不要修改其他页面的 HTML。
- 不要删减 `payload_budget` 要求的承载内容来"解决"过挤问题。
- 不要用 `position: absolute + transform: translate(...)` 布局流式内容。
- 不要用 `mt-auto`、`flex-grow` 或 `flex: 1` 伪造间距。
- 不要把所有组件退回成纯色背景 + 白卡片。
- 不要为了"简洁"删掉 motif 装饰层。
