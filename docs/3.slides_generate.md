## PPT 工程化链路（PPT Engineering Pipeline）

目标：把“对话式生成 PPT”升级为**可拆解、可回放、可校验、可回滚的工程化流水线**。
核心思想：**以 Skill 为最小执行单元，以 Event 为状态记录，以 Artifact 为可版本化产物。**

---

# 一、总体流程结构（Stateful Pipeline）

整体不是“一次性 prompt 生成 PPT”，而是一个**多阶段、可中断、可迭代的状态机流程**。

```
材料导入 → 结构理解 → 大纲生成 → 大纲确认
    ↓
逐页内容生成 → HTML 结构化 → 资源补齐（图/表/数据）
    ↓
渲染校验 → 交互修改 → 再生成
    ↓
版式映射 → PPTX 工程化转换 → 导出
```

每一步：

* 都有独立 Skill
* 都产出 Artifact（可版本化）
* 都可人工干预
* 都可恢复重跑

---

# 二、状态机设计（Agent Loop 决策逻辑）

建议为 PPT 生成建立**显式任务状态机**：

```text
INIT
↓
COLLECTING_MATERIAL
↓
OUTLINE_DRAFTED
↓
OUTLINE_CONFIRMED
↓
PAGE_GENERATING
↓
PAGE_RENDER_CHECK
↓
READY_TO_EXPORT
↓
EXPORTED
```

Agent Loop 每轮根据：

* 当前 Session State
* 已有 Artifact
* 用户操作
* 校验结果
* 成本预算

来判断下一步调用哪个 Skill。

# 三、阶段拆解（Skill 分层）

## 1️⃣ 材料收集与结构化（Material Processing）

### 输入来源

* URL
* PDF
* Word
* 邮件
* 本地文件夹
* 对话补充

### 核心 Skill

| Skill                | 作用                         |
| -------------------- | -------------------------- |
| `material.ingest`    | 下载/解析/落盘                   |
| `material.parse`     | 提取结构（标题/段落/表格/图）           |
| `material.normalize` | 去噪、去广告、Prompt Injection 降权 |
| `material.index`     | 建立检索索引                     |

### 产物（Artifact）

```
materials/
  raw/
  parsed.json
  normalized.json
  embeddings/
```

### 技术要点

* 外部内容必须先进入结构化 JSON
* 不允许直接把网页全文拼进 prompt
* 所有材料必须可追溯来源

---

## 2️⃣ 大纲生成（Outline Planning）

### Skill

* `ppt.generate_outline`
* `ppt.rewrite_outline`
* `ppt.merge_outline`
* `ppt.estimate_page_count`

### 输入

* 用户目标（主题/受众/时长）
* 材料索引
* 用户历史偏好（长期记忆）

### 输出 Artifact

```
artifacts/
  outline_v1.json
  outline_v2.json
```

### 设计原则

* 大纲必须是结构化 JSON（而不是自然语言段落）
* 每页必须有：

  * page_id
  * title
  * key_points[]
  * source_refs[]
  * suggested_visual_type（图/表/示意图）

### 用户交互

UI 提供：

* 拖拽排序
* 增删页
* 重命名
* 插入“空白自定义页”

Agent 只负责更新结构，不直接覆盖用户改动。

---

## 3️⃣ 逐页内容生成（Content → HTML）

这是工程化的核心。

### Skill

* `ppt.generate_page_content`
* `ppt.generate_page_html`
* `ppt.rewrite_page`
* `ppt.expand_point`
* `ppt.summarize_page`

### 数据流

```
outline.json
    ↓
page_content.json
    ↓
page_html.html
```

### 强制分离

**绝不能直接 LLM 生成最终 PPTX**

必须分层：

1. 语义层（内容）
2. 表达层（HTML）
3. 渲染层（PPTX）

---

## Page JSON 示例

```json
{
  "page_id": "p3",
  "title": "市场规模",
  "bullets": [
    {
      "text": "2025 年市场规模达 500 亿",
      "importance": "high",
      "source_ref": "doc_3"
    }
  ],
  "visual": {
    "type": "bar_chart",
    "data_ref": "dataset_2"
  }
}
```

---

## HTML 生成规则

* 使用受控模板
* 禁止自由 CSS
* 所有结构可被程序解析

例如：

```html
<section data-page-id="p3">
  <h1>市场规模</h1>
  <ul>
    <li>2025 年市场规模达 500 亿</li>
  </ul>
  <div data-visual="bar_chart"></div>
</section>
```

# 四、资源生成与增强（Visual & Asset Pipeline）

## 图片与图表

### Skill

* `image.search`
* `image.generate`
* `chart.generate`
* `chart.validate`

### 策略

* 优先搜索
* 不满足再生成
* 所有图片必须记录来源或生成参数

Artifact:

```
assets/
  images/
  charts/
  meta.json
```

---

# 五、渲染效果校验（Render Check Loop）

这是工程质量的关键模块。

### Skill

* `ppt.render_check`
* `ppt.layout_validate`
* `ppt.overflow_detect`
* `ppt.font_missing_check`

### 校验内容

* 文本溢出
* 图片变形
* 标题过长
* 字号不统一
* 列表层级异常

若失败：

→ 自动回到 `rewrite_page`
→ 或提示用户人工调整

---

# 六、HTML → PPTX 工程化转换

这是区别“玩具生成”与“生产级系统”的核心。

### 转换流程

```
HTML AST
    ↓
Layout Mapping Engine
    ↓
Slide Object Model
    ↓
PPTX XML
```

### 技术实现建议

* 建立自己的 Slide Schema
* 映射到 PowerPoint XML 结构
* 或通过 Office 自动化 API 写入

### Skill

* `ppt.map_layout`
* `ppt.export_pptx`

### 输出

```
final_v1.pptx
```

# 七、版本管理与回滚（Artifact Versioning）

每一步都生成版本节点：

```
outline_v3
page_p2_v5
html_p2_v4
pptx_v2
```

支持：

* 对比差异
* 回滚
* 局部重生成

---

# 八、Agent Loop 决策逻辑示例

伪逻辑：

```python
if state == OUTLINE_DRAFTED and not user_confirmed:
    wait_user()

elif state == OUTLINE_CONFIRMED:
    next_page = find_unfinished_page()
    call(ppt.generate_page_content)

elif state == PAGE_RENDER_CHECK_FAILED:
    call(ppt.rewrite_page)

elif all_pages_passed and not exported:
    call(ppt.export_pptx)
```

# 九、成本与性能优化策略

* Outline 阶段允许大模型
* Page 阶段可降级模型
* HTML 生成使用小模型
* 图表生成走模板逻辑
* Cache embeddings
* 增量生成而非全量重算

---

# 十、为什么这是“工程化链路”

区别于普通 LLM 生成：

| 普通方式       | 工程化方式              |
| ---------- | ------------------ |
| 一次性 prompt | 分阶段 Skill          |
| 无结构输出      | JSON → HTML → PPTX |
| 不可回滚       | 版本链                |
| 不可校验       | Render Check       |
| 黑盒         | 事件可回放              |
