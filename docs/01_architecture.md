# AgentOS 架构设计文档 v0.1

## 项目概述

AgentOS 是一个基于事件驱动架构的 AI Agent 开发平台，提供类 VS Code 风格的交互界面。本版本专注于实现最核心的对话和工具调用能力，为后续扩展奠定基础。

## 核心设计理念

### 事件驱动架构
系统采用事件总线模式，各模块通过发布/订阅事件进行解耦通信。这种设计使得系统具备良好的可扩展性和可维护性。

### 双总线机制
- **Public Bus**: 全局事件总线，用于系统级模块间通信
- **Private Bus**: 每个 Agent 实例拥有独立的私有总线，确保会话隔离

Bus Router 模块负责根据 `session_id` 将公共事件路由到对应 Agent 的私有总线。

## 技术栈选型

### 前端技术栈
- **框架**: Next.js + TypeScript
- **状态管理**: React Context API
- **通信协议**: 原生 WebSocket
- **UI 组件**: shadcn/ui + Tailwind CSS
- **样式风格**: VS Code 风格的深色主题

### 后端技术栈
- **Web 框架**: FastAPI (Python 3.12)
- **WebSocket**: 原生 websockets 库
- **异步框架**: asyncio
- **事件总线**: 内存队列实现
- **数据存储**: SQLite
- **配置管理**: YAML 格式

### LLM 集成
- **支持的提供商**: OpenAI、Anthropic
- **响应模式**: 非流式（v0.1 版本）
- **模型选择**: 固定模型（v0.1 版本）

## 数据流转示例

用户发送消息的完整流程：

1. 用户在前端输入消息
2. WebSocket 发送 `ui.user_input` 事件到后端
3. Gateway 将事件发布到 Public Bus
4. Bus Router 根据 `session_id` 路由到对应 Agent 的 Private Bus
5. AgentRuntime 接收事件，发布 `agent.step_started`
6. ContextBuilder 构建 LLM 输入，发布 `llm.call_requested`
7. LLM Module 调用 API，发布 `llm.call_completed`
8. AgentRuntime 解析响应，如需工具调用则发布 `tool.call_requested`
9. ToolRuntime 执行工具，发布 `tool.call_completed`
10. 循环直到完成，发布 `agent.step_completed`（目前循环结束条件为：LLM 返回的结果中不再包含 tool call）
11. 结果通过 WebSocket 返回前端展示

## 本地存储策略

### 存储根目录

存储根目录由配置文件中的 `storage.root` 字段指定，默认值取决于运行模式：

| 模式 | 默认存储根目录 |
|------|---------------|
| 开发模式（`dev`） | `./SenseAssistant`（当前工作目录） |
| 正式模式（`production`） | `~/.SenseAssistant`（用户 home 目录） |

### 会话数据存储
每个会话的工作目录：`<storage.root>/workspace/<session_id>/`

### 日志存储
- 系统日志：`<storage.root>/logs/system.log`
- 会话日志：`<storage.root>/workspace/<session_id>/session.log`

### 数据库位置
- 主数据库：`<storage.root>/agentos.db`

## 安全与权限

### v0.1 版本策略
- **文件访问**: 无限制（全系统访问）
- **命令执行**: 直接执行，无沙箱
- **工具超时**: 统一 15 秒
- **用户认证**: 暂不实现

> ⚠️ 注意：v0.1 版本为开发测试版本，生产环境需要增强安全策略。

## 模块优先级

### 核心模块（必须实现）
1. Event Bus（事件总线）
2. AgentRuntime（Agent 运行时）
3. LLM Module（LLM 集成）
4. ToolRuntime（工具运行时）

### 支撑模块
5. WebSocket Gateway
6. ContextBuilder
7. Bus Router
8. Database Layer

### 前端模块
9. 对话界面
10. WebSocket 客户端
11. 文件浏览器
12. 历史记录
