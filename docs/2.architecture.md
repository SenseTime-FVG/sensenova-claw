## 架构设计（Architecture）

### 目标与范围

本系统是一个运行在**本地电脑**上的自动化 AgentOS，形态上参考 openclaw 一类“本地可控的电脑自动化代理”，核心应用场景聚焦办公生产力，尤其是 **PPT 工程化生成链路**：

多渠道输入（网页/Doc/PDF/邮件/本地文件/对话）→ 自动生成大纲 → 逐页生成 HTML 内容 → 可视化交互修改 → 工程化转换为 **可编辑 PPTX**

本文件回答
- **系统怎么分层、每层负责什么**


### 分层结构（Layered Architecture）

整体采用“底层可控工具 + 上层智能编排”的分层结构。底层提供可控执行与治理，上层负责推理、规划与交互体验。

- **Driver Layer** 提供“可执行、可控、可审计”的工具与事件总线，是整个系统的最小可信底座。
- **System Support Layer** 像 OS 的“安全/审计/资源管理”子系统一样，贯穿所有层，统一治理。
- **Agent Runtime** 管理会话、工作区、沙箱与并行编排，为 Agent Kernel 提供稳定运行时。
- **Agent Kernel** 专注能力与策略：规划、记忆、工具使用、主动交互、多 Agent 通信。
- **应用层 UI** 通过事件总线/本地 API 与 Agent 交互，实现“生成—编辑—再生成—导出”的闭环体验。

### Driver Layer（工具与执行层）

目标：把“控制电脑/处理资料/产出文件”抽象为**标准化工具调用**，并做到**安全可控**。

执行模式：

- **本地直连执行**：直接调用本机 API/驱动（效率高，风险高，需要更严格的权限与审计）。
- **沙箱执行**：在隔离环境运行（风险低，可控性强，适合处理不可信网页/PDF/脚本）。

关键组件：

- **Event Bus**
  - 统一事件协议：UI 输入、Agent 决策、tool call 请求/响应、进度、错误、人工确认等
  - 支持订阅/发布、回放（用于崩溃恢复与可复现）
- **Tool Runtime**
  - 统一工具注册与发现（tool schema/权限/成本/超时/重试策略）
  - 工具调用的生命周期管理：pre-check → 执行 → post-check → 产出落盘 → 事件记录
- **Sandbox Manager（在 Driver 或 Runtime 中均可实现，推荐由 Runtime 管理、Driver 提供能力）**
  - 创建/销毁沙箱、挂载目录、网络策略、进程/CPU/内存限制
  - GUI/无 GUI 两种形态（浏览器自动化通常需要 GUI 或 headless）
- **Local Drivers**
  - CLI、文件系统、剪贴板、截图、键鼠、Office 自动化、浏览器自动化等
- **Agent Process Manager**
  - Agent 进程隔离、心跳、崩溃重启、资源限制


补充说明：**事件总线（Event Bus）** 不仅用于 UI↔Agent 通信，也用于把“工具调用生命周期”“策略拦截/确认”“产物版本点”“错误与恢复动作”统一事件化，形成可回放的事件流。事件命名与建议事件类型见后文「事件总线与交互协议」。

### System Support Layer（系统治理层）

目标：像 OS 的“安全子系统/审计/资源管理”一样，统一治理，保证可控、可追溯、可运营。

#### 1）日志与可观测（Observability）

- **事件日志（Event Log）**：用户输入、Agent 决策、工具调用、结果、错误、人工确认、导出动作
- **Tracing**：每个 Session 一个 `trace_id`
- **Metrics**：latency、token、tool 成功率、PPT 生成时长、重试次数、失败原因分布
- **产物留痕**：每次生成/修改形成版本号，可回滚（见 Artifact Versioning）

#### 2）成本与资源治理（Cost & Resource Governor）

- **预算系统**：session budget / 用户限额 / 组织限额（可选）
- **并发控制**：multi-agent 并发上限；工具调用限流；避免“并行爆 token”
- **缓存**：LLM 调用缓存、解析缓存（PDF/网页抓取）、检索缓存、PPT 生成中间产物缓存

#### 3）安全模块（Security & Policy Engine）

- **身份**：本地用户/设备绑定；（可选）SSO/OAuth；（可选）mTLS
- **授权**：网站/邮箱/企业 IM 的 OAuth/Token 管理（密钥保存在本地安全存储）
- **最小权限**：文件访问白名单、网络域名白名单、危险操作二次确认（键鼠控制、删除文件、发送邮件等）
- **Prompt Injection 防护**：对外部输入（网页/PDF/邮件）进行降权与结构化提取，工具调用前强制策略校验

### Agent Runtime Environment（运行环境层）

目标：为 Agent Kernel 提供“像操作系统进程一样”的稳定运行时：会话、工作区、并行编排、恢复与隔离。

关键组件：

- **Workspace Manager**
  - 管理工作区目录结构（materials / artifacts / cache / logs）
  - 负责导入来源材料的落盘、去重、索引（供检索/引用）
- **Session Manager**
  - 多 Session 并行、Session 状态机（running / waiting_user / blocked_policy / failed / completed）
  - 崩溃恢复：基于 Event Log 与 Artifact 版本链恢复到最近稳定点
- **Sandbox Module**
  - 选择何时进入沙箱：不可信输入、需要下载/执行脚本、需要隔离浏览器操作等
- **Multi-agent Orchestrator**
  - 负责并行任务拆解：检索/解析/大纲/逐页生成/质量检查/图表生成等
  - 合并结果与冲突处理（页面级合并、引用与来源对齐）
- **Channel Gateway（可选）**
  - 企业微信/飞书/Slack/Teams/邮件等多渠道入口适配
- **Tool Manager（工具注册与治理）**
  - 工具注册
  - 工具权限管理
  - 执行前校验
  - 执行后落盘

### Agent Capability Layer（Agent Kernel / 能力层）

目标：“任务理解—规划—工具调用—交互—迭代”，核心Loop。

建议的 Kernel 能力切分：

- **Memory**
  - 短期上下文管理（本次 Session 的任务状态、引用材料）
  - 长期记忆（用户偏好、常用模板、企业术语库；需可控开关与可擦除）
  - 压缩摘要（用于降低 token 与提高检索效率）
- **Tool-use Policy**
  - 工具选择
  - 失败重试策略、回退策略
- **Skills（技能系统）**
  - 将“可复用的任务流程/最佳实践”封装为技能（例如 `ppt.generate_outline`、`ppt.generate_page_html`、`ppt.render_check`、`ppt.export_pptx`），由 Agent Kernel 在 loop 中按需调用与组合，而不是把所有逻辑固化在单次 prompt 中。
- **Communication（多 Agent 协作）**

### Application Layer（应用层 / UI）

目标：让用户在“生成—编辑—再生成—导出”闭环中保持控制感与可预期性。

UI/交互核心能力：

- 任务创建：选择输入源（URL/PDF/Word/对话/文件夹）
- 大纲确认：结构可视化，支持用户改写/插页/删页
- 逐页 HTML 编辑：所见即所得编辑 + 与 Agent 联动的“建议/改写/替换图表”
- 一键导出 PPTX：导出前检查（字体/版式/资源缺失），导出后可本地打开
- 追溯与回滚：每次生成/编辑形成版本点，可对比与回滚

### PPT 工程化链路

利用系统组件(上下文管理、长短期记忆、交互tool、skill)作为技术支撑，直接通过skill来构建ppt生成

agent loop中，自动判断当前应当执行的步骤
主要行为包括 ppt大纲生成，当前页内容生成，当前页html生成，图片搜索，图片生成，html修改，渲染效果校验 等

### 事件总线与交互协议

事件总线用于统一承载 **UI↔Agent↔Tool** 的通信、进度与状态变更，并将“决策—执行—产物—质量—恢复”全流程事件化，形成**可回放的事件流**（崩溃恢复、对比回滚、问题定位、可观测统计）。

对齐参考：事件模型建议与 `pi-mono` 的“agent/turn/message/tool”分层保持一致（同一条用户请求对应一个或多个 turn；每个 turn 内包含 0..N 个 tool span；消息流式更新用 message_update）。

#### 事件命名与层级

参考pi-mono

- **命名规范**：`<domain>.<action>`，domain 建议固定枚举：`ui/agent/tool/artifact/policy/budget/sandbox/error/session`。
- **层级语义（Tracing）**：
  - **Agent 级**：`agent.start` / `agent.end`（会话内长期运行或一次任务生命周期）
  - **Turn 级**：`turn.start` / `turn.end`（一次“接收输入→做出决策→产出回复/动作”的循环）
  - **Message 级**：`message.start` / `message.update`（仅助手流式）/ `message.end`
  - **Tool 级**：`tool.execution_start` / `tool.execution_update` / `tool.execution_end`
  - 以上层级通过 `trace_id`、`span_id`、`parent_span_id` 串联（turn span 通常是 tool span 的 parent）

#### 核心事件类型（建议最小集合）

- **UI 事件**：`ui.user_input`、`ui.selection_changed`、`ui.confirmed`、`ui.edited_html_saved`
- **Agent 决策事件**：`agent.plan_created`、`agent.step_started`、`agent.step_completed`、`agent.request_user_confirmation`
- **工具调用事件**：`tool.call_requested`、`tool.call_started`、`tool.call_result`（也可与 `tool.execution_*` 统一/映射）
- **产物事件**：`artifact.created`、`artifact.updated`、`artifact.version_checkpointed`、`artifact.exported_pptx`
- **系统治理事件**：`policy.denied`、`policy.require_confirmation`、`budget.exceeded`、`rate_limited`、`sandbox.created`
- **错误与恢复事件**：`error.raised`、`error.recovered`、`session.state_changed`

#### 事件封装（实现时固化 schema）

- **必备字段**：
  - `event_id`：全局唯一（用于去重与幂等）
  - `ts`：时间戳（毫秒）
  - `type`：事件类型（如 `tool.execution_end`）
  - `session_id`：会话标识
  - `trace_id`：一次任务/链路追踪标识
  - `span_id`：当前事件所属 span
  - `parent_span_id`（可选）：用于构建层级关系
  - `actor`：`ui | agent | tool | system`
  - `payload`：事件负载（建议可扩展、但保持类型内稳定）
- **通用 payload 建议字段**（按需）：
  - `name`（tool/skill/step 名称）、`status`（running/succeeded/failed/cancelled）
  - `input_ref` / `output_ref`（指向工作区文件路径或对象存储 key，避免把大内容塞进事件）
  - `error`（code/message/retriable/stack_ref）
  - `metrics`（latency_ms/token/tool_cost 等）

#### 投递语义与一致性约束

- **投递语义**：建议至少一次（at-least-once），消费者幂等（用 `event_id` 去重）。
- **顺序保证**：同一 `session_id`（或同一 `trace_id`）内尽量保持顺序；若无法强顺序，至少保证同一 `span_id` 内单调递增（用 `ts` + `seq`）。
- **落盘策略**：关键事件（工具开始/结束、artifact checkpoint、policy 决策、error）必须落盘；大内容走 `*_ref` 引用。
- **回放恢复**：以 `artifact.version_checkpointed` 作为稳定点，回放时重建 session/turn 状态机并恢复到最近稳定点。


### 安全与隐私（本地优先）

与 PRD 对齐的原则：

- **所有文件默认不上传云端**；如需调用远程模型，必须显式提示与可配置开关
- **文件访问白名单**：仅允许访问用户选择/授权的目录或文件
- **工具调用权限隔离**：按工具类别/风险分级（读取、写入、网络、键鼠、Office 控制等）
- **沙箱隔离**：对不可信输入与高风险动作优先沙箱执行
