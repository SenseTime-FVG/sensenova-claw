# 贡献者行为准则

## 我们的承诺

作为项目成员、贡献者和维护者，我们承诺让每一个参与者都能获得无骚扰的体验——无论年龄、体型、残障状况、民族、性别认同与表达、经验水平、国籍、外貌、种族、宗教信仰或性取向如何。

## 我们的标准

**有助于营造良好环境的行为包括：**

- 使用友善和包容的语言
- 尊重不同的观点和经验
- 积极接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

**不可接受的行为包括：**

- 使用带有性暗示的语言或图像，以及不受欢迎的性关注
- 故意挑衅、侮辱性评论，以及人身或政治攻击
- 公开或私下骚扰
- 未经明确许可发布他人的个人信息（如地址、电子邮箱）
- 其他在专业环境中被合理认为不当的行为

## 维护者的责任

项目维护者有责任为可接受的行为标准做出诠释，并对任何不可接受的行为采取恰当且公正的纠正措施。

维护者有权删除、编辑或拒绝与本准则不符的评论、提交、代码、Wiki 编辑、Issue 和其他贡献，也有权暂时或永久封禁任何表现出不当、威胁、冒犯或有害行为的贡献者。

## 适用范围

本准则适用于所有项目空间，包括但不限于：

- GitHub 仓库（Issues、Pull Requests、Discussions、代码评审）
- 项目相关的社交媒体和通讯渠道
- 线上或线下的项目活动

当个人在公共场合代表本项目或社区时，本准则同样适用。

## 执行

遇到辱骂、骚扰或其他不可接受的行为，请通过以下方式举报：

- **GitHub Issue**：在仓库中提交私密安全报告
- **邮件**：联系项目维护者（见 [GitHub 组织主页](https://github.com/SenseTime-FVG)）

所有投诉都将被审查和调查，并以维护者认为必要且恰当的方式作出回应。维护者有义务对举报者信息保密。

## 执行指南

维护者在判定违反行为准则的后果时，将参考以下指南：

### 1. 提醒

**适用于**：首次轻微违规，如无意中使用不当语言。

**后果**：维护者私下提醒，解释该行为为何不当，并要求改正。

### 2. 警告

**适用于**：单次较严重的违规，或多次轻微违规。

**后果**：发出正式警告，要求在一定期限内避免与相关人员互动。违反此条件可能导致暂时或永久封禁。

### 3. 暂时封禁

**适用于**：严重违规，或在警告后继续不当行为。

**后果**：在指定期限内禁止参与项目的一切互动和公开交流。

### 4. 永久封禁

**适用于**：持续违反准则，骚扰个人，或对特定群体的攻击行为。

**后果**：永久禁止参与项目的一切公开互动。
