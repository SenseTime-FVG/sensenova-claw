
# Agent 交互界面设计需求文档 (2026年3月版)

## 1. 项目概述
本项目旨在开发一个基于 Electron 框架，采用类 Visual Studio Code (VS Code) 风格的现代化 AI Agent 交互界面。目标是为开发者提供一个高度专注、可定制且功能强大的 AI 协作环境。

## 2. 界面架构设计 (VS Code 风格)
借鉴 VS Code 的三栏式布局，界面将分为五个核心区域：
- **标题栏 (Title Bar)**: 集成 AI 原生搜索框（类似 Command Center）及窗口控制。
- **活动栏 (Activity Bar)**: 位于最左侧，放置切换“资源管理器”、“聊天历史”和“插件扩展”的图标。
- **主侧边栏 (Primary Side Bar)**: 承载当前活动视图的具体内容。
- **中央交互区 (Main Stage)**: 核心 AI 聊天界面及多模态预览区。
- **状态栏 (Status Bar)**: 显示当前 Agent 状态、Token 消耗、运行环境及模型版本。

## 3. 侧边栏功能需求 (Primary Side Bar)

### 3.1 资源管理器 (Explorer)
- **文件树状结构**: 支持本地项目目录的可视化，允许选择文件并将其作为上下文直接拖拽至聊天区。
- **打开的项目**: 显示当前 AI 正在阅读或修改的文件列表。
- **大纲视图**: 展示当前选定代码文件的函数与类结构，方便定位。

### 3.2 聊天历史 (Chat History)
- **归档列表**: 按时间（今天、近七天、更早）分类存储对话记录。
- **搜索与筛选**: 支持通过关键词、使用的模型节点或特定项目标签搜索历史会话。
- **分支管理**: 记录每一次对话的分支路径，允许用户回到之前的任何节点重新发起对话。

## 4. 中央聊天界面布局

### 4.1 对话流设计
- **泡泡/流式布局**: AI 回复支持实时流式渲染（Streaming），支持 Markdown 高级语法、LaTeX 公式及 Mermaid 流程图。
- **代码块组件**:
    - 支持 2026 年主流语言的高亮显示。
    - **快捷操作**: 提供“一键应用到本地文件”、“复制”及“在终端运行”按钮。
- **多模态预览**: 图片、PDF、数据图表等直接在聊天流内提供交互式预览。

### 4.2 输入区域 (Input Area)
- **命令面板模式**: 输入 `/` 弹出快捷指令（如 `/refactor`, `/test`）。
- **智能提及**: 输入 `@` 调起上下文引用（文件、文件夹、Git 提交记录）。
- **附件管理**: 支持通过拖拽或粘贴方式添加跨媒体素材。

## 5. Electron 实现特殊考虑

### 5.1 性能与系统集成
- **原生外观**: 在 macOS/Windows 上使用 `frame: false` 和 `titleBarStyle: 'hiddenInset'` 实现沉浸式标题栏。
- **多进程架构**: UI 渲染进程与 AI 逻辑处理进程隔离，确保在处理大型 context 时界面不卡顿。
- **本地存储**: 使用 SQLite 或 LevelDB 本地化存储长期的聊天历史和大模型缓存。

### 5.2 窗口管理
- **多窗口支持**: 允许用户将预览页面或文档拆分到独立窗口。
- **系统通知**: 耗时较长的 Agent 任务通过系统通知反馈结果。

## 6. 视觉与交互规范 (2026 趋势)
- **主题系统**: 提供 `Dark+` (默认)、`Light+` 及高对比度主题。支持基于 JSON 的自定义色彩配置。
- **字体标准**: 默认采用等宽字体 `Fira Code` 或 `JetBrains Mono` 以获得最佳代码平衡。
- **微交互**: 图标切换时的动态反馈，加载状态的呼吸灯效果。

## 7. 结论
通过复刻 VS Code 的分层布局与扩展性逻辑，结合 Electron 的系统能力，该界面将能为用户提供一个既熟悉又高效的 AI 协作体验。
