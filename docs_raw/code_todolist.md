### 双总线机制
- **Public Bus**: 全局事件总线，用于系统级模块间通信
- **Private Bus**: 每个 Agent 实例拥有独立的私有总线，确保会话隔离

Bus Router 模块负责根据 `session_id` 将公共事件路由到对应 Agent 的私有总线。

当前架构实际是什么样的
所有模块 → PublicEventBus（广播）→ 所有订阅者收到所有 session 的所有事件
每个 Runtime 自己在事件处理函数里做 session_id 过滤（通过 state_store.get_turn(session_id, turn_id)），靠业务逻辑来实现隔离，而不是靠总线层的物理隔离。

潜在风险
目前多个 session 并发时，AgentRuntime 会收到所有 session 的事件
靠 state_store 的 key 来避免串扰。这在单实例场景下能工作，但没有物理隔离，调试困难
